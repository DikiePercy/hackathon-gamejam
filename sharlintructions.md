# sharlintructions

## 1) Быстрая карта проекта

Проект собран вокруг трех игровых контуров:
- меню и управление сохранениями,
- основной рейс (бой на поезде),
- депо (апгрейды и подготовка к следующему рейсу).

Базовый цикл:
1. Меню -> запуск рейса.
2. Рейс в main: спавн волн, бой, таймер, проверка условий успеха/провала.
3. Завершение рейса -> переход в депо.
4. Депо: трата золота, апгрейды, возврат в рейс.

## 3) Связи между файлами

### Main <-> GameManager
- main.gd читает параметры миссии из GameManager.gd.
- main.gd по окончанию миссии вызывает apply_mission_result и autosave_current_state.
- main.gd при смерти игрока пытается загрузить latest save через pending_load_data.

### Main <-> Train
- main.gd слушает сигналы поезда:
  - go_trein,
  - stop_trein,
  - train_objective_broken.
- main.gd берет из train.gd:
  - get_total_alive_passengers,
  - get_train_integrity.

### Train <-> Wagon
- train.gd строит вагоны из GameManager.train_data.
- wagon.gd спавнит пассажиров по seat markers.
- train.gd суммирует живых пассажиров через методы вагонов.

### Train <-> Enemy
- train.gd запускает волны врагов, создает экземпляры enemy.
- enemy.gd получает цели посадки через setup_for_train_raid.
- enemy.gd может наносить урон объекту поезда через damage_train.

### Player <-> Main/Pause
- main-person.gd испускает died, который ловит main.gd.
- pause.gd управляет паузой и запускает save/load через GameManager.

### Depot <-> GameManager
- depot.gd читает и меняет total_gold, total_p, has_shotgun, train_data.
- depot.gd применяет pending_mission_reward после возвращения с рейса.

## 4) Что допиливать в первую очередь

### Блок A: Устойчивый цикл рейса
Сделать стабильным полный сценарий:
menu -> main -> depot -> main и обратно с сохранением прогресса.

Что делать:
1. Проверить, что все переходы сцен идут через единые константы путей.
2. Проверить, что save/load в pause.gd не конфликтует с autosave в main.gd.
3. Добавить единый вывод причин fail/success и наград в UI.

Файлы:
- scenes/main/main.gd
- scenes/main/pause.gd
- scenes/GameManager/GameManager.gd

### Блок B: Экономика и апгрейды
Закрыть баланс между рейсом и депо.

Что делать:
1. Ввести таблицу цен и эффекта апгрейдов (урон, hp, вместимость, скорость).
2. Привязать апгрейд вагона к фактическим статам в wagon.gd и train.gd.
3. Добавить мягкие ограничения, чтобы игрок не ломал баланс ранним snowball.

Файлы:
- scenes/depot/depot.gd
- scenes/wagon/wagon.gd
- scenes/train_timur/train.gd
- scenes/GameManager/GameManager.gd

### Блок C: Боевой ИИ и читаемость боя
Сделать поведение врагов понятным и предсказуемым.

Что делать:
1. Вынести ключевые пороги enemy.gd в отдельный блок констант баланса.
2. Добавить debug-режим (опционально): отрисовка дистанций detection/shoot/stop.
3. Проверить, что враги корректно выбирают цель: пассажир -> игрок -> поезд.

Файлы:
- scenes/characters/enemy.gd
- scenes/train_timur/train.gd

### Блок D: UX сохранений
Довести систему слотов до удобного состояния.

Что делать:
1. Показать текущий слот в меню/паузе.
2. Показать timestamp сейва и результат последнего рейса.
3. Разделить быстрый autosave и ручные save-слоты в UI.

Файлы:
- scenes/menu/node_2d.gd
- scenes/main/pause.gd
- scenes/GameManager/GameManager.gd

## 5) Практический план внедрения (короткий)

Шаг 1:
Стабилизировать переходы и сейвы.
- Сначала main.gd + pause.gd + GameManager.gd.
- Прогон: смерть игрока, load, переход в депо, назад в main.

Шаг 2:
Доработать поезд/вагоны.
- Прогон: спавн пассажиров, урон по поезду, падение поезда в fail.

Шаг 3:
Доработать депо.
- Прогон: покупка, апгрейд, влияние на следующий рейс.

Шаг 4:
Доработать enemy AI и тесты.
- Прогон: волны, поведение на лестницах/крыше, урон по целям.

## 6) Минимальный Definition of Done

Фича считается готовой, если:
1. Есть рабочий цикл main -> depot -> main без ручных фиксов в редакторе.
2. Save/load восстанавливает состояние без поломки позиции/здоровья/поезда.
3. Апгрейды реально меняют геймплей на следующем рейсе.
4. Нет критических ошибок в консоли Godot при 5-10 мин тестового забега.

## 7) Где чаще всего ломается

- Сигналы между сценами (особенно при переименовании методов).
- Неправильный индекс вагона при апгрейде в депо.
- Расхождение состояния между локальными переменными сцены и GameManager.
- Загрузка сейва в сцену с отличающейся структурой узлов.
